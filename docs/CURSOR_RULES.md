# Cursor AI Rules

## Project Overview

This project is the official portfolio website of **Yagya Prasad Sapkota**, an aspiring IT student passionate about Artificial Intelligence, Data Science, Cloud Computing, and Cybersecurity.

The goal is to build a premium, modern, scalable, and production-ready portfolio using industry best practices.

---

# Tech Stack

- React 19
- TypeScript
- Vite
- Tailwind CSS v4

Only use these technologies unless explicitly instructed otherwise.

Never install additional packages without permission.

---

# General Rules

- Build one feature at a time.
- Never modify unrelated files.
- Never delete existing functionality unless instructed.
- Keep code clean and modular.
- Prefer readability over cleverness.
- Avoid unnecessary complexity.

---

# React Rules

- Use Functional Components only.
- Use TypeScript for every component.
- Use named props interfaces.
- Avoid inline functions when unnecessary.
- Keep components focused on a single responsibility.

---

# Component Rules

Each component should:

- Have one clear responsibility.
- Be reusable whenever possible.
- Remain reasonably small and easy to understand.
- Avoid duplicated logic.

---

# Styling Rules

Use only Tailwind CSS.

Do not:

- write inline styles
- create unnecessary CSS files
- use random colors
- hardcode spacing values inconsistently

Maintain a consistent design system.

---

# Code Quality

Write code that is:

- readable
- maintainable
- reusable
- production-ready

Avoid:

- duplicated code
- unused imports
- unused variables
- unnecessary comments

---

# Accessibility

Always:

- use semantic HTML
- add aria-labels where appropriate
- support keyboard navigation
- maintain good color contrast

Accessibility is required.

---

# Responsiveness

Every component must work on:

- Mobile
- Tablet
- Laptop
- Desktop

Mobile-first design is preferred.

---

# Performance

Avoid unnecessary re-renders.

Use lazy loading only when needed.

Do not optimize prematurely.

---

# Folder Structure

Respect the existing architecture.

Do not create new folders unless requested.

Place every file in the correct location.

---

# Naming Convention

Components:

PascalCase

Example:

Navbar.tsx

Hero.tsx

Button.tsx

Files:

PascalCase for components

camelCase for utility files

---

# Imports

Keep imports organized.

Remove unused imports.

Use consistent ordering.

---

# Error Handling

Never ignore TypeScript errors.

Fix them correctly.

Do not suppress errors.

---

# AI Behavior

If requirements are unclear:

Do not guess.

Instead, explain what information is missing.

Never invent:

- projects
- achievements
- certifications
- experience
- education

Only use information provided by the user.

---

# Output Rules

When generating code:

- Explain what files were created.
- Explain why each file exists.
- Do not modify unrelated files.
- Keep explanations concise.

---

# Project Philosophy

This portfolio should feel:

- Professional
- Modern
- Developer-focused
- Minimal
- Premium

Quality is more important than speed.
